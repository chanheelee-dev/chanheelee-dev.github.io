# Specification: Initialize Conductor Management for Blog

## Overview
This track formalizes the management of the personal blog `chanheelee-dev.github.io` using the Conductor methodology. It ensures that the project structure, content management, and deployment workflows are documented and aligned with the project's vision.

## Objectives
- Document existing Jekyll structure (Essays, Projects).
- Verify and document the dual-hosting strategy (GitHub Pages for production, Netlify for previews).
- Establish a consistent workflow for adding new content.

## Key Components
- **Content**: Essays (`content/essays/`), Projects (`content/projects/`).
- **Static Site Generation**: Jekyll (Ruby-based).
- **Deployment**: GitHub Pages (main), Netlify (preview).

## Success Criteria
- Conductor configuration accurately reflects the current project state.
- Tracks Registry and Implementation Plan are correctly initialized.
- Deployment workflow is verified and documented in the workflow guide.
